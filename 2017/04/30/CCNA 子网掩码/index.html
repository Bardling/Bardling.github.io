CCNA 子网掩码

Apr 30, 2017 | CCNA | Hits

子网掩码把网络分为各种子网，也可以把小的网络合并成大的网络。但是子网掩码必须结合 IP 地址一起使用，常用的 IP 地址分为 A 类、B 类、C 类。三种类别分别对应不同范围的子网掩码，A 类子网掩码第一字节为255，B 类子网掩码以255.255打头，而 C 类子网掩码以255.255.255打头

C 类网络的子网划分

在 C 类地址中只有 8 位用于定义主机。

但是不能使用 /31 和 /32，因为至少要 2 个主机位。

子网划分要熟悉五个问题

  • 选定的子网掩码将创建多少个子网？
  • 每个子网可包含多少台主机？
  • 每个子网的广播地址是什么？
  • 每个子网的广播地址是什么？
  • 每个子网可包含哪些主机地址？

举一个例子来进行示范
网络地址 = 192.168.10.0
子网掩码 = 255.255.255.128
255.255.255.128的CIDR值为 /25, /25 代表子网掩码为25位的1，即255.255.255.128，写为二进制便是前三子节全部为1，第四子节为1000000。

  • 计算多少个子网？ 2^x，x 为取值为 1 的数。例如这一题
    就是为 2 的 1 次方，子网数为 2。
  • 每个子网多少台主机？ 2^y - 2，y 为取值为 0 的数，但是减 2 是因为要拿出两个作为子网地址和广播地址。例如这一题主机数量就为2^7 - 2 = 126。
  • 有哪些合法子网？ 先用 256 - 子网掩码，算出来的数为一个块。例如之一题，256 - 128 = 128，所以块的大小就为128，合法的子网以一个块的大小来分割，直到达到子网掩码值。所以这题的合法子网为 0 和128。
  • 每个子网的广播地址是什么？ 在下一个子网之前的数字，所有主机位都为1，就是当前子网的广播地址。在这个例子中，对于子网0，广播地址就为127 。
  • 每个子网包含多少哪些合法主机地址？ 在子网地址和广播地址之间的数字就是合法的主机。这一个例子中的合法主机就为 1 ～ 126 和 129 ～ 254 之间的所有地址都为合法主机，0 和 128 用作了子网地址，127 和 255 用作广播地址。

B 类网络的子网划分

B 类网络的计算方式和 C 类一样，但是 B 类的使用的主机位更多，而且子网掩码不仅仅可以在第四子节划分，还可以在第三子节。

在 B 类网络中，子网号和广播地址都用两个子节表示，例如 128.255。

示例1
网络地址 = 172.16.0.0
子网掩码 = 255.255.128.0
首先，255.255.128.0 的 CIDR 为 /17 。

  • 子网掩码第三子节为 1000 0000 ，所以子网数为2^1 = 2
  • 第三子节有 7 个 0，再加上第四子节的 8 个，所以每个子网的主机数为2^15 - 2 =32766
  • 256 - 128 =128 ，因此两个合法的子网为 0.0 和 128.0 。
  • 每个子网的广播地址为下一个子网地址 - 1，所以子网 0.0 的广播地址为127.255。128.0 的广播地址为 255.255。
  • 合法的主机地址就为子网地址和广播地址之间的所有地址。0.0 子网的主机地址为 0.1 ～ 127.254 ，128.0 的主机地址为 128.1 ～ 255.254。

示例1计算的是子网掩码在第三子节的情况，还有一种情况是子网掩码在第四子节的。/24 和 /25 是比较特殊的两个B类子网掩码，现在来分别计算这两个。

示例2
网络地址 = 172.16.0.0
子网掩码 =255.255.255.0 （/24）
B 类 /24 的后两子节为1111 1111 . 0000 0000。

  • 子网数量为2^8 = 256
  • 每个子网的主机数量为2^8 - 2 = 254
  • 合法的子网为 256 -255 = 1 ，所以子网为 0、1、2 、······ 、255
  • 广播地址为 0.255、1.255 、2.255、······ 、255.255
  • 主机地址为 0.1 ～ 0.254、 1.1 ～ 1.254 、2.1 ～ 2.254、······、255.254

示例3
网络地址 = 172.168.0.0
子网掩码 = 255.255.255.128 （/25）
/25是比较难处理的子网掩码。第三、四子节为 1111 1111 . 1000 0000 。

  • 子网数为 2^9 = 512
  • 每个子网的主机数为 2^7 - 2 =126
  • 合法的子网数为 256 - 255 = 1，因此第三字节取值数为 0、1、2、3 等，但是子网数为 512 个，每个子网主机数 126 个，第三子节只有 0 ～255 个，所以第三子节每一个取值分为两个块，例如第三子节取值为 1 时，对应两个子网为 1.0 和 1.128
  • 例如第三子节取 1 时，广播地址分为两个子网，1.0 的广播地址为 1.127，1.128 的广播地址为 1.255
  • 合法的主机地址就为子网地址和广播地址之间

A 类网络的子网划分

A 类网络的子网划分和 B 类和 C 类没有什么区别 ，只是处理位数增加到了24位，而 B 类和 C 类分别是 16 位和 8 位。

示例4
255.255.0.0 (/16)
/16 第二三四子节为，1111 1111 . 0000 0000 . 0000 0000 。

  • 子网数为 2^8 = 256
  • 每个子网的主机数为 2^16 - 2 = 65534
  • 256 - 255 = 1 ，因此子网为 10.0.0.0 、10.1.0.0 、10.2.0.0、······ 、10.255.0.0
  • 每个子网的广播地址为 10.0.255.255 、10.1.255.255、10.2.255.255、······ 、10.255.255.255
  • 合法主机地址一样为子网地址和广播地址之间

示例5
255.255.240.0 (/20)
/20 的第二三四位为，1111 1111 . 1111 0000 . 0000 0000 。

  • 子网数为 2^12 = 4096
  • 每个子网的主机数为 2^12 - 2 =4094
  • 在第二子节中，256 -255 = 1，但在第三子节中 256 -240 =16。所以在第二子节中子网分隔为 1，在第三子节子网为 0 、16 、32 等。例如 第二子节为 1 时，子网为 10.1.0.0 、10.1.16.0 、10.1.32.0 等。
  • 以下略

示例6
255.255.255.192 (/26)
/26的第二三四子节为，1111 1111 . 1111 1111 . 1100 0000。

  • 子网数为 2^18 = 262 144
  • 每个子网的主机数为 2^6 - 2 = 62
  • 合法子网在第二和第三子节中，子网间隔为 1，在第四子节中子网间隔为 256 - 192 = 64
  • 以下略
